## 🚀 更新步骤

### 步骤 1：备份当前配置（推荐）

在 Home Assistant 中：
```yaml
# 备份当前卡片配置
type: custom:amap-commute-card
entity: sensor.xxx
amap_web_key: your_key
title: 今日通勤
```

### 步骤 2：更新前端卡片文件

#### 方法 A：通过文件管理器更新

1. 找到以下文件：
   ```
   /config/www/amap-commute-card.js
   ```

2. 备份旧文件（可选）：
   ```bash
   cp /config/www/amap-commute-card.js /config/www/amap-commute-card.js.bak
   ```

3. 上传新文件：
   - 将 `www/amap-commute-card.js` 上传到 `/config/www/`
   - 覆盖原文件

#### 方法 B：通过 Home Assistant 文件编辑器更新

1. 打开 Home Assistant
2. 进入 **设置 → 文件编辑器**
3. 打开 `/config/www/amap-commute-card.js`
4. 替换整个文件内容为新版本代码
5. 保存

### 步骤 3：清除浏览器缓存

#### Web 界面
1. 按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新
2. 或打开开发者工具（F12），右键点击刷新按钮，选择"清空缓存并硬性重新加载"

#### Home Assistant App（iOS/Android）

**iOS：**
1. 关闭 Home Assistant App
2. 打开系统设置 → Home Assistant
3. 向下滚动，找到"清除缓存"或"重置"
4. 重新打开 App

**Android：**
1. 打开系统设置 → 应用 → Home Assistant
2. 存储 → 清除缓存
3. 重新打开 App

### 步骤 4：测试更新

## 🐛 常见问题

### Q1: 更新后卡片显示空白或异常

**原因**：浏览器缓存了旧版本

**解决方案**：
1. 清除浏览器缓存（步骤 3）
2. 强制刷新（Ctrl + Shift + R）
3. 如果问题依旧，重启 Home Assistant

### Q2: 手机端地图仍然显示异常

**原因**：App 缓存未清除

**解决方案**：
1. 完全关闭 Home Assistant App
2. 清除 App 缓存（步骤 3）
3. 重新打开 App
4. 如果问题依旧，卸载重装 App

### Q3: 地图加载失败

**原因**：amap_web_key 配置问题或网络问题

**解决方案**：
1. 检查 `amap_web_key` 是否正确配置
2. 检查网络连接
3. 查看浏览器控制台错误信息
4. 参考 `README.md` 中的配置说明

### Q4: 点击功能不工作

**原因**：JavaScript 错误或版本不匹配

**解决方案**：
1. 打开浏览器控制台（F12）
2. 查看是否有 JavaScript 错误
3. 确认 `amap-commute-card.js` 已完整上传
4. 检查文件大小是否正常（约 30KB）
